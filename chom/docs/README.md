## Documentation Standards

### File Naming Conventions

- `README.md` - Overview/introduction for a section
- `*-GUIDE.md` - Step-by-step instructional guides
- `*-QUICK-REFERENCE.md` - Quick lookup references
- `*-SUMMARY.md` - Executive summaries
- `*-IMPLEMENTATION.md` - Implementation details
- `*-CHECKLIST.md` - Checklists and verification lists

### Writing Guidelines

1. **Be concise** - Get to the point quickly
2. **Use examples** - Show, don't just tell
3. **Link related docs** - Help readers navigate
4. **Keep it current** - Update as code changes
5. **Follow the style guide** - See [../CODE-STYLE.md](../CODE-STYLE.md)
